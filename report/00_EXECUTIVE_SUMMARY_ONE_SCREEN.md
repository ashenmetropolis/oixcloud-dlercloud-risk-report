# 先看这页：oixCloud / 原 DlerCloud 风险速览

> 本页用于让新用户在 30 秒内看懂风险。完整证据见 `evidence/`、`docs/chat_redacted/` 和 `report/`。

## 一句话结论

**不建议新用户购买 oixCloud 高价长期套餐；已有用户不建议把它作为唯一主力服务。**

原因不是单个用户不会配置，而是材料中同时出现了：

```text
DlerCloud 改名/承接痕迹
+ oixCloud 公告长期救火
+ 官方端/访问控制/封端争议
+ 多客户端 Unauthorized / 订阅失败 / Timeout
+ 流量异常暴涨
+ 工单/客服响应不足
+ 高价套餐权益争议
```

---

## 7 个最直观风险

| 风险 | 用户能看到的现象 | 影响 |
|---|---|---|
| 1. 接盘/换壳风险 | DlerCloud → oixCloud 改名/承接特征明显 | 不是全新稳定起盘，旧问题可能延续 |
| 2. 官方端绑定/封端风险 | 第三方客户端、路由器、自定义配置可用性下降 | 用户失去客户端自由和故障自救能力 |
| 3. Unauthorized 鉴权风险 | 更新后无法登录、添加配置 Unauthorized | 不是节点慢，而是账号/订阅/鉴权体系异常 |
| 4. 订阅与配置风险 | QX、Surge、Shadowrocket、OpenClash 等导入失败 | 多平台用户无法稳定使用 |
| 5. 节点与线路风险 | 节点全红、Timeout、节点减少、IEPL/Fusion 异常 | 实际可用性不稳定 |
| 6. 流量统计风险 | 一晚几百 G、一天上 T、关代理仍异常消耗 | 套餐剩余价值和高速额度可能被快速耗尽 |
| 7. 售后响应风险 | 工单多日未回复、客服缺位 | 出问题后用户只能自己互相求助 |

## 对不同用户的建议

### 新用户

**不建议入。**  
尤其不建议买年付、高价、钻石、Premium、长期套餐。

### 已有低价用户

可作为备用，但不建议作为唯一主力。

### 技术用户 / 路由器用户

尤其谨慎。  
如果你依赖 Surge、sing-box、OpenClash、Shadowrocket、Quantumult X、Clash Verge、自定义 DNS、软路由或旁路由，官方端绑定和封端会明显降低可控性。

### 高价套餐用户

重点保留：

- 套餐截图；
- 付款记录；
- 公告截图；
- Unauthorized / Timeout / 流量异常截图；
- 工单未回复截图；
- 节点减少、权益变化截图。

```text
DlerCloud 公告/频道资料
→ oixCloud 改名/承接特征

→ 结论：高价长期套餐风险明显，不建议重仓
```
