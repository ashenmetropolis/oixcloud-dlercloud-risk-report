# 01 证据充足性判断 v3

## 总结

新上传的 `oixcloud问题反馈.docx` 比上一版更全，证据强度明显提高。它把“群聊讨论”进一步落到“故障现场截图 + 用户反馈窗口”层面。

当前证据足够支撑以下公开避雷主线：

> oixCloud 具备 DlerCloud 改名/承接特征；接手后持续出现订阅、DNS、客户端、协议、线路、访问控制、封端/官方端依赖、售后响应、流量统计和高价套餐权益争议。对新用户，尤其高价长期套餐用户，不建议重仓或作为唯一主力。

## 能稳妥证明

1. 多客户端、多平台集中异常；
2. Unauthorized/鉴权异常高频出现；
3. 订阅、配置、导入、DNS、协议兼容问题广泛存在；
4. Snell / AnyTLS / Shadowrocket / Quantumult X / Surge / OpenClash / Clash Verge 等兼容问题形成集群；
5. 工单/客服/售后响应不足；
6. 流量统计异常是重大新增风险；
7. 高价/长期套餐的实际体验与预期不匹配；
8. DlerCloud → oixCloud 的延续盘风险需要提示。

## 不建议写死

- “一定诈骗”
- “所有用户都被封端”
- “所有用户都流量异常”
- “Dler 原群踢人证据完整”

## 公开表述边界

为避免夸大和误导，本项目不使用“一定诈骗”“所有用户都被封端”“所有用户都流量异常”等绝对化定性。

更稳妥的表述是：存在集中反馈、存在明显争议、存在高风险特征、存在服务稳定性与长期套餐权益不确定性，证据足够用于购买前避雷提示。

本项目定位为公开资料整理与风险提示，不替代司法、仲裁或平台最终裁定。
