# 03 可公开版结论 v3

## oixCloud / 原 DlerCloud 用户风险提示

根据 DlerCloud 公告、oixCloud 公告、oixCloud 群聊导出和新版 `oixcloud问题反馈.docx`，可以看到 oixCloud 具备明显 DlerCloud 改名/承接特征。接手后，服务长期出现订阅、DNS、客户端、协议、线路、访问控制和官方端依赖问题。

新版问题反馈窗口进一步显示，大量用户集中反馈：

- Unauthorized / 未授权；
- 订阅失败；
- 节点全红；
- Timeout；
- Shadowrocket、Quantumult X、Surge、OpenClash、Clash Verge、FlClash 等客户端异常；
- Snell / AnyTLS 兼容错误；
- 工单多日无回复；
- 流量异常暴涨；
- Premium / Bronze / 699 / 长期套餐权益争议。

这些内容表明，oixCloud 的问题并非单个用户配置错误，而是多个平台、多个客户端、多个用户同时出现的结构性体验问题。

尤其需要注意的是，封端/官方端绑定会削弱用户对第三方客户端、自定义配置和路由器环境的控制权；流量异常则直接影响套餐剩余价值和高速额度；售后响应不足会让用户在故障后缺少有效处理路径。

因此，本记录建议：

1. 新用户不要购买高价长期套餐；
2. 已有用户不要把 oixCloud 作为唯一主力；
3. 技术用户、路由器用户、第三方客户端用户尤其谨慎；
4. 高价套餐用户应保留公告、套餐、问题反馈和流量异常截图。
